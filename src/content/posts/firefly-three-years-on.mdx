Firefly was a collaborative real-time debugger for distributed Ruby systems.
It worked. It also did a lot of work to do something that, three years later,
the rest of the ecosystem now does for free.

Here's what I'd change.

## Don't reinvent the protocol

We wrote a custom WebSocket protocol for stepping a paused process and brokering
control between connected clients. Most of the bugs lived in there.

Today I'd start from the [Debug Adapter Protocol](https://microsoft.github.io/debug-adapter-protocol/)
and treat collaboration as a thin layer on top of it. DAP is well-understood,
has a real spec, and gives you VSCode/JetBrains/etc. integration the day you
ship.

The lesson isn't "DAP is great." It's "if there's a published spec for the
boring core of your problem, your differentiator should not be replacing the
spec."

## Treat Redis like a cache

We treated Redis like a database — sessions, ACLs, transcripts, all in there.
It's a cache. The minute we needed to ask a real question about a session
("which engineer paused this most recently?"), we ended up writing query code
against a key-value store and pretending it was a relational one.

Postgres for the truth, Redis for the things that genuinely die when a process
restarts. That's it.

## Don't optimize the demo path

The demo was beautiful: two engineers stepping through a live session,
annotating frames, handing off control. The 95th-percentile real flow was
*one* engineer, alone, reproducing a bug in staging, and our UX assumed the
multi-player case from the first paint.

If I rebuilt it I'd start solo and earn the multi-player UI by being asked for
it. Three of every four "feature gaps" in the early backlog were
single-player concerns we'd cut to chase the demo.

## What I still believe

The premise — that production debugging is a *team* activity that we keep
modeling as a solo one — still holds. Slack threads where someone copies a
backtrace into a code block and three people argue about what `nil` means in
context, that's the actual workflow. A tool that absorbs that has room.

I just wouldn't be the one to write it again from scratch. I'd fork an
existing DAP client and start there.
